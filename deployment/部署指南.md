## 🖥️ 环境要求

### 操作系统
- **Linux** (Ubuntu 18.04+, CentOS 7+, Debian 9+)
- **macOS** (10.14+)

### 必须安装的软件

| 软件 | 最低版本 | 推荐版本 | 用途 |
|------|---------|---------|------|
| Node.js | v18.0.0 | v20.x LTS | JavaScript 运行时 |
| npm | v9.0.0 | v10.x | Node.js 包管理器 |

### 安装 Node.js

#### macOS 安装方式

**方式一：使用 Homebrew（推荐）**
```bash
# 安装 Homebrew（如果没有）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Node.js
brew install node

# 验证安装
node -v
npm -v
```

**方式二：使用安装包**
- 下载地址：https://nodejs.org/
- 选择 LTS 版本（推荐）或 Current 版本
- 下载后双击安装包，按照提示完成安装

#### Linux 安装方式

**方式一：使用包管理器（Ubuntu/Debian）**
```bash
# 更新软件源
sudo apt update

# 安装 Node.js 和 npm
sudo apt install nodejs npm

# 验证安装
node -v
npm -v
```

**方式二：使用 nvm（推荐）**
```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重新加载 shell 配置
source ~/.bashrc

# 安装 Node.js LTS 版本
nvm install --lts

# 验证安装
node -v
npm -v
```

**方式三：使用 NodeSource 仓库**
```bash
# 添加 NodeSource 仓库（以 Node.js 20.x 为例）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# 安装 Node.js
sudo apt install nodejs

# 验证安装
node -v
npm -v
```

## 🚀 快速开始

### 第一步：进入部署目录

```bash
cd frontend/deployment
```

### 第二步：设置执行权限

```bash
chmod +x deploy.sh
```

### 第三步：检查环境

```bash
./deploy.sh --check
```

如果所有检查都通过，您将看到：
```
[SUCCESS] Node.js 版本检查通过
[SUCCESS] npm 版本检查通过
[SUCCESS] 依赖已安装
[SUCCESS] 所有检查通过！
```

### 第四步：启动服务

**开发模式（推荐用于开发）**
```bash
./deploy.sh --dev
```

**生产模式（用于正式部署）**
```bash
./deploy.sh --prod
```

服务启动后，访问：http://localhost:5173

## 📖 详细使用说明

### 部署脚本选项

| 选项 | 说明 | 示例 |
|------|------|------|
| `--check` | 只进行依赖和版本检查 | `./deploy.sh --check` |
| `--build` | 只构建项目 | `./deploy.sh --build` |
| `--dev` | 启动开发服务器 | `./deploy.sh --dev` |
| `--prod` | 构建并启动生产服务器 | `./deploy.sh --prod` |
| `--stop` | 停止运行的服务 | `./deploy.sh --stop` |
| `--clean` | 清理构建文件 | `./deploy.sh --clean` |
| `--help` | 显示帮助信息 | `./deploy.sh --help` |
| `--version` | 显示版本信息 | `./deploy.sh --version` |

### 完整部署流程

```bash
# 1. 检查环境
./deploy.sh --check

# 2. 构建项目（可选，开发模式不需要）
./deploy.sh --build

# 3. 启动服务
./deploy.sh --dev    # 开发模式
# 或
./deploy.sh --prod   # 生产模式
```

### 修改配置

配置文件位于：`config/app.conf`

```bash
# 修改端口
APP_PORT=8080

# 修改最低 Node.js 版本要求
NODE_MIN_VERSION=20.0.0
```

## ❓ 常见问题

### Q1: 提示 "Node.js 版本过低"

**问题**：脚本拒绝执行，提示 Node.js 版本太低

**解决方案**：
1. 卸载旧版本 Node.js
2. 安装新版本（v18.0.0 或更高）

```bash
# macOS 卸载
sudo rm -rf /usr/local/bin/npm /usr/local/share/man/man1/node* /usr/local/lib/dtrace/node.d ~/.npm ~/.nve ~/.cache/node

# Linux 卸载
sudo apt remove nodejs npm
sudo apt autoremove
```

### Q2: 端口被占用

**问题**：提示 "端口 5173 已被占用"

**解决方案**：

方法一：停止占用端口的服务
```bash
./deploy.sh --stop
```

方法二：修改配置文件中的端口
```bash
# 编辑 config/app.conf
APP_PORT=8080
```

方法三：查找并停止占用端口的进程
```bash
# macOS
lsof -ti:5173 | xargs kill -9

# Linux
fuser -k 5173/tcp
```

### Q3: 依赖安装失败

**问题**：`npm install` 执行失败

**解决方案**：

1. 清除 npm 缓存
```bash
npm cache clean --force
```

2. 删除 node_modules 和 package-lock.json
```bash
rm -rf node_modules package-lock.json
```

3. 重新安装
```bash
npm install
```

### Q4: 构建失败

**问题**：`npm run build` 执行失败

**解决方案**：

1. 检查是否有语法错误
```bash
npm run lint
```

2. 检查 TypeScript 编译错误
```bash
npx tsc --noEmit
```

3. 清理并重新安装依赖
```bash
./deploy.sh --clean
npm install
npm run build
```

## 🔧 故障排除

### 查看详细错误信息

```bash
# 使用 bash -x 调试模式运行
bash -x ./deploy.sh --check
```

### 检查 Node.js 和 npm 版本

```bash
node -v    # 应显示 v18.0.0 或更高
npm -v     # 应显示 9.0.0 或更高
```

### 检查端口占用情况

```bash
# macOS
lsof -i:5173

# Linux
ss -tuln | grep 5173
# 或
netstat -tuln | grep 5173
```

### 清理所有缓存

```bash
# 清理 npm 缓存
npm cache clean --force

# 清理 node_modules
rm -rf node_modules

# 清理构建文件
rm -rf dist

# 清理 Vite 缓存
rm -rf .vite

# 重新安装
npm install
```

### 查看日志

如果服务启动失败，检查是否有错误输出。

### 验证安装

```bash
# 验证 Node.js
node -e "console.log('Node.js works!')"

# 验证 npm
npm -v

# 验证项目依赖
npm list
```

## 📝 注意事项

- 部署脚本会自动检查 Node.js 和 npm 版本，版本过低将无法执行
- 生产部署建议使用 `--prod` 选项
- 开发环境建议使用 `--dev` 选项
- 确保端口 5173 未被占用，或在配置文件中修改端口
- 首次部署会自动安装依赖，后续部署会跳过已安装的依赖
